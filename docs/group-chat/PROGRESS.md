# Group Chat Progress

## 当前分支
- 工作分支：`codex/group-chat-phase1`
- 基线来源：`main` / `0.1.3`
- 当前状态：已暂停，暂不并回 `main`

## 本阶段目标
- 建立群聊会话模型
- 支持创建群、邀请、移除、改名、退群、解散
- 支持群内消息发送与群维度历史搜索
- 支持 `@` 点名与迟到回复基础提示
- 不影响当前主线 `main`

## 当前已完成
- 服务端已扩展为统一 conversation 模型：
  - 单 agent 会话
  - 群聊会话
- 新增群聊存储与接口：
  - 建群
  - 群详情
  - 打开群
  - 改群名
  - 邀请成员
  - 移除成员
  - 退群
  - 解散群
  - 群历史分页
  - 群历史搜索
  - 会话快照
- 群聊后端采用“每群每 agent 一条独立上游 session”。
- 同一 agent 在同一群内采用串行队列处理用户消息。
- 群内支持 `@` 点名，发送时会把 mention 元数据传给服务端。
- 群消息历史支持：
  - 群系统消息
  - 群成员发言人显示
  - 迟到回复标记
  - 回应目标摘要
- 前端已接入：
  - 左侧群聊卡片
  - 左侧新建群聊入口
  - 头部群聊管理入口
  - 设置面板中的群聊区
  - 群聊创建/管理面板
  - 群成员管理操作
  - 群内 `@` 候选菜单
  - 群维度搜索复用
  - 已退出/已解散群的只读查看入口

## 已做验证

## 当前限制
- 完整 `npm run selftest` 仍可能在上游对话等待阶段耗时较长，不适合作为本阶段唯一阻断项。
- 群聊 UI 虽已接入，但仍需要一轮真实浏览器人工验收，重点检查：
  - 群聊创建和管理交互
  - `@` 候选选择体验
  - 群内迟到回复展示
  - 退群和解散后的只读查看体验
- “未被 @ 的 agent 自主决定是否发言”当前依赖群聊 bootstrap 协议约束，仍需继续观察实际效果。
