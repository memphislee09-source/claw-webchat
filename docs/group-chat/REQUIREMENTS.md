# Group Chat Requirements

## 产品形态
- 群聊是左侧列表中的独立会话类型，与单 agent 会话并列显示。
- 群聊卡片当前优先显示：
  - 群名
  - 成员数
  - 最新消息摘要
  - 最近活跃时间
- 头像区域第一期先使用简化群图标，不做多头像拼图。

## 创建与管理
- 用户可以创建空群。
- 用户也可以在创建群时直接邀请若干 agent。
- 群名为必填项。
- 第一阶段需要支持：
  - 创建群
  - 邀请 agent
  - 移除 agent
  - 改群名
  - 用户退群
  - 解散群
- 新建群入口放在左侧主入口，设置面板中也要提供群聊管理入口。

## 成员与上下文
- 群成员范围为 `用户 + 多个 agent`。
- 群内所有 agent 共享同一条本地群时间线。
- 后端按“每群每 agent 一条独立上游 session”建模。
- 新加入的 agent 不补看进群前完整历史，只从加入时刻开始参与后续上下文。
- agent 被移出群后：
  - 历史消息保留
  - 不再接收后续群消息
  - 不再显示为当前成员

## 发言规则
- 每次用户发送消息时，消息会广播给当前群成员 agent。
- 如果用户 `@某个 agent`，该 agent 必须回复一次。
- 如果一条消息 `@` 多个 agent`，这些被点名的 agent 都必须回复。
- 未被 `@` 的 agent 自主决定是否发言。
- 如果不发言，界面不显示“未参与本轮”之类提示，保持安静。
- agent 不自动连续讨论。
  - 它们只响应“用户新消息”这一轮。
  - 不因为其他 agent 的消息而继续无限接话。

## @ 机制
- 群聊第一阶段支持 `@agent`。
- 展示层按 `@显示名` 呈现。
- 内部识别以 `agentId` 为准。
- 允许重名。
- 如果存在重名，必须通过候选列表显式选中目标成员，不能靠纯文本盲匹配。

## 并发与迟到回复
- 同一群里，同一个 agent 采用串行队列。
- 如果 agent A 还在回复上一条用户消息，此时用户又发了下一条：
  - 新消息先进入该 agent 在本群的队列
  - 等上一条处理完成后再顺序处理
- 用户在 agent 仍在回复时可以继续发送新消息。
- 群时间线按真实完成时间推进。
- 迟到回复定义为：
  - 某条 agent 回复归属较早一轮用户消息
  - 但到达时群里已经出现了更新的用户消息
- 迟到回复规则：
  - 仍按真实完成时间插入
  - 不回插旧位置
  - 要显示“迟到回复”弱提示
  - 还要标明它回应的是哪一轮用户消息

## 系统消息
- 以下事件需要进入群历史，作为可见系统消息：
  - 创建群聊
  - 邀请 agent 入群
  - 移除 agent
  - 用户退群
  - 解散群
  - `/new` 重置上下文

## /new 语义
- 群聊支持 `/new`
- 语义为：
  - 重置该群所有当前成员 agent 的上游上下文
  - 保留本地群历史
  - 插入 `已重置上下文` 标记

## 搜索
- 群聊支持历史搜索。
- 搜索范围仅限当前打开的群聊时间线。
- 搜索结果至少包含：
  - 发言人名称
  - 时间
  - 摘要/片段
  - 点击跳转
- 点击后需要高亮命中关键词。
- 最近搜索记录按“群聊维度”保存。

## 媒体能力
- 群聊默认沿用现有单聊媒体能力。
- 用户继续可发送：
  - 图片
  - 音频
- agent 继续可返回：
  - 图片
  - 音频
  - 视频
  - 文件

## 状态与归档
- 用户退群后，该群从左侧列表移除。
- 群解散后：
  - 从左侧列表移除
  - 本地历史保留
  - 可从设置菜单调出只读查看
